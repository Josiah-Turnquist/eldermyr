Creature art — current vs upgraded

Every creature is drawn live in canvas code; there are no art files. Each is rendered at its true game size then upscaled nearest-neighbour — so what you see is exactly what survives at real scale. Shown on their actual in-game ground, because pop on a neutral swatch means nothing. Both columns run the real shipped code (current = git HEAD, upgraded = working tree) over the game's real tile renderer — nothing here is a re-drawing.

Recoloured variants — same branch, different e.color

Colour is data: the same code paints a Skeleton bone-white and a Dread warlord red. Every tone in the upgrade derives from e.color, so these keep their identity. The old boss hardcoded its robe, which is why The Pale Shepherd was violet.

What changed: same silhouette, same flat-colour language — plus a dark rim so each creature reads on grass and on dungeon stone, two tones of shading for form, one highlight, and motion that sells weight. Nothing became unrecognisable; the slime is still a blob, just a good one.

On the boss: the previous version of this page compared a hand-copied “current” boss that was missing its head and crown (18 ops — the real branch is 29). Both columns now run the real code, so the crown is back in view. The upgraded boss keeps the crown and the e.color head deliberately: they are the bright focal point that stops a dark robe reading as a blob on the dungeon floor, and the head is the only thing that makes The Pale Shepherd pale.

Ops = canvas call-sites in the branch (static count, both flash and non-flash paths). Measured cost at 131 creatures: the art is 3.05× the old — but the draw loop never culled off-screen enemies, and adding that cull makes the shipping frame 0.598 ms → 0.363 ms, i.e. faster than before despite the extra detail.
